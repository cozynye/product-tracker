# 📈 중고 아이템 시세 분석 서비스 (PriceTracker) 기획서

중고나라, 번개장터 등 주요 중고 거래 플랫폼의 데이터를 크롤링하여 시세 추이를 시각화하고, 사용자가 설정한 적정가에 맞는 매물을 추적하는 서비스입니다.

---

## 1. 기술 스택

- **Framework**: Next.js 15 (App Router, TypeScript)
- **Database**: Supabase (인증 없음 — 단일 사용자 도구)
- **Styling**: shadcn/ui + Tailwind CSS
- **Visualization**: Recharts
- **Crawling**: Next.js API Routes
  - 번개장터: REST API 직접 호출 (`api.bunjang.co.kr`)
  - 중고나라: Cheerio + `__NEXT_DATA__` JSON 파싱
  - 참조: `/Users/sinsunyeong/Desktop/ref/junggo/src/scraper.js`

## 2. 주요 기능 및 페이지 상세

### 2.1 메인 페이지 (Dashboard)

- **등록된 검색어 리스트**: 사용자가 등록한 아이템들을 카드 형태로 나열.
- **검색어 등록 (모달)**: `shadcn/ui`의 Dialog 컴포넌트 활용.
  - **입력 필드**:
    - 검색어 (필수)
    - 카테고리 (고정: 시계, 슈프림)
    - 적정가 (비필수)
    - 알림가 (비필수)
    - 제외 키워드 (비필수, 콤마로 구분)

### 2.2 상세 분석 페이지 (Analysis Detail)

- **시세 요약**: 메인 검색어, 현재 평균가, 나의 적정가 표시.
- **시세 추이 그래프 (Recharts)**:
  - 최대 6개월간의 일별/월별 판매 가격 추이.
  - **적정가 라인**: 그래프 내 가로 점선으로 표시하여 시각적 비교 제공.
- **데이터 필터 컨트롤**:
  - 플랫폼 선택: 중고나라, 번개장터 (토글 버튼).
  - 상태 필터: 판매중, 예약중, 거래완료 (멀티 체크박스).
  - 제외 키워드 관리: 실시간 추가/삭제 및 즉시 그래프/리스트 반영.
- **아이템 리스트**:
  - **Hot Deals**: '판매중' 상태이면서 '적정가' 이하인 매물 최상단 배치.
  - **전체 리스트**: 모든 크롤링 데이터 및 수동 입력 데이터를 날짜순 정렬.
  - **상세 이동**: 각 아이템 클릭 시 원본 게시글 URL로 이동.
- **수동 데이터 입력**: 사용자가 직접 거래 날짜와 가격을 입력하여 그래프에 반영.

## 3. 데이터베이스 설계 (Supabase)

### 3.1 `user_monitors` (검색어 설정 테이블)

> 인증 없는 단일 사용자 도구이므로 `user_id` 컬럼 없음.

| Column              | Type        | Description                         |
| :------------------ | :---------- | :---------------------------------- |
| `id`                | uuid (PK)   | 고유 식별자                         |
| `keyword`           | text        | 메인 검색어                         |
| `category`          | text        | 시계 / 슈프림                       |
| `target_price`      | int8        | 사용자 설정 적정가                  |
| `alert_price`       | int8        | 알림 기준 가격 (향후 카카오 알림용) |
| `excluded_keywords` | text[]      | 필터링할 키워드 배열                |
| `last_crawled_at`   | timestamptz | 마지막 크롤링 시각 (3시간 캐시 기준) |
| `created_at`        | timestamptz | 생성일                              |

### 3.2 `price_snapshots` (시세 데이터 테이블)

> `(monitor_id, url)` 복합 unique constraint — 동일 매물 중복 저장 방지.

| Column       | Type        | Description                          |
| :----------- | :---------- | :----------------------------------- |
| `id`         | uuid (PK)   | 고유 식별자                          |
| `monitor_id` | uuid (FK)   | user_monitors 참조                   |
| `platform`   | text        | `joonggonara` / `bunjang` / `manual` |
| `title`      | text        | 매물 제목                            |
| `price`      | int8        | 판매 가격 (원)                       |
| `status`     | text        | `판매중` / `예약중` / `거래완료`     |
| `url`        | text        | 원본 게시글 주소 (unique key)        |
| `posted_at`  | timestamptz | 게시글 등록 일자                     |
| `created_at` | timestamptz | DB 저장 시각                         |

## 4. 핵심 구현 로직

1. **크롤링 아키텍처**:
   - **번개장터**: `https://api.bunjang.co.kr/api/1/find_v2.json` REST API 직접 호출.
     - 파라미터: `q=검색어&n=100&order=score&page=0`
     - 응답: `list[].{ pid, name, price, status("0"=판매중), update_time(Unix ts) }`
     - 상품 URL: `https://bunjang.co.kr/products/{pid}`
   - **중고나라**: `https://web.joongna.com/search/{keyword}` HTML 페이지 요청 후 Cheerio로 `#__NEXT_DATA__` JSON 추출.
     - 응답 경로: `props.pageProps.dehydratedState.queries[i].state.data.data.items`
     - 상품 URL: `https://web.joongna.com/product/{seq}`
   - **병렬 실행**: `Promise.all([bunjang, joonggonara])`으로 동시 호출.
   - 참조 구현: `/Users/sinsunyeong/Desktop/ref/junggo/src/scraper.js`

2. **크롤링 캐시 정책**:
   - `user_monitors.last_crawled_at` 기준 **3시간 이내 재방문 시 DB 캐시 반환**.
   - 상세 페이지에 **"새로고침" 버튼** 제공 → 클릭 시 캐시 무시하고 즉시 재크롤링.
   - 신규 데이터는 `(monitor_id, url)` unique constraint 기반 **upsert**로 저장.

3. **필터링 프로세스**:
   - `excluded_keywords` 리스트를 순회하며 제목에 포함된 매물은 원천 배제.
   - UI에서 선택된 플랫폼/상태값에 따라 클라이언트 사이드에서 즉시 리렌더링.

4. **적정가 분류**:
   - `price <= target_price` && `status === '판매중'` 조건 충족 시 Hot Deals 섹션 배치.
